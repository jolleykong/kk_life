# 场景六：普通索引列条件，上S锁，再向表内尝试插入大于记录已有最大值的记录。

## 场景

| t1                                                  | t2                                                           |
| --------------------------------------------------- | ------------------------------------------------------------ |
| begin ;<br>select * from t1 where c3 = 7 for share; | begin ;                                                      |
|                                                     | insert into t1 select 12,12,12,12;<br>c3是辅助索引，c3=7记录不存在，是否阻塞？ |

### 结果：<font color=red>阻塞，有没有索引都阻塞。RR级别上S锁后，无论记录是否存在都会有gap lock，gap lock 和插入意向锁不兼容，因此对同一个gap进行插入动作会被阻塞。</font>
